# Unity 美术通用工具合集

Unity 美术通用工具合集是一个专为美术、关卡、运营等非程序同学设计的 **Unity 编辑器工具面板**。  
通过简单配置，即可把日常高频操作集中到一个窗口里，一键执行，大幅减少来回点击和重复劳动。

> 仓库地址：[https://github.com/yuxuhai/UnityArtTool](https://github.com/yuxuhai/UnityArtTool)

---

## 适用人群与场景

- **适用人群**
  - 场景 / 角色 / 特效美术
  - 关卡设计、数值策划、运营同学
  - 需要经常在多个场景、资源、工具菜单之间来回切换的任何成员

- **典型使用场景**
  - 一键打开常用场景（主城、战斗、登录、测试场景等）
  - 一键执行常用编辑器菜单（烘焙、打 AB、导出资源等）
  - 快速定位项目中的资源或场景中的物体
  - 集中管理项目 Wiki / 打包平台 / 后台等网页入口
  - 为团队成员提供“傻瓜式”的操作面板

---

## 核心功能一览

- **自定义标签页工具面板**
  - 支持多个标签页（Tab），每个标签页可配置任意数量的工具项
  - 标签页可拖拽排序、重命名、删除

- **丰富的内置工具项类型**
  - `ToolAsset`：执行 Unity 菜单命令（如 Lightmap 烘焙、打包命令等）
  - `FindObjectAsset`：在 Project 窗口中定位指定资源
  - `FindGameObjectAsset`：在场景中查找指定名称的 GameObject
  - `TextAsset`：显示说明文字 / 操作步骤
  - `OpenPathAsset`：一键打开系统文件夹（如截图目录、打包输出目录）
  - `OpenSceneAsset`：一键切换场景，支持保存当前修改
  - `OpenWebAsset`：打开常用网页 / 后台 / 文档地址
  - `SeparatorAsset`：分隔符，用于视觉分组，提升可读性

- **编辑模式 + 拖拽排序**
  - 开启“编辑模式”后，可添加 / 删除工具项
  - 支持对标签页、工具项进行拖拽排序，所见即所得

- **快速搜索区域**
  - 内置搜索入口：Google / 百度 / Bing / GitHub / StackOverflow / Unity 文档 / 知乎
  - 在编辑器内输入关键字，一键跳转到选定搜索引擎

- **联机启动与全局设置**
  - 可设置“联机启动场景”，底部一键启动并自动 Play

## 安装与打开

### 方式一：通过 Unity 包管理器（推荐）

> 前提：本仓库已经作为一个独立 Git 仓库托管在 GitHub 上。

1. 在目标项目中打开 **Package Manager**：`Window -> Package Manager`。
2. 点击左上角 `+` 按钮，选择 **Add package from git URL...**。
3. 输入仓库地址：
   - `https://github.com/yuxuhai/UnityArtTool.git`
4. 等待 Unity 下载并导入完成后，即可在菜单栏通过 `工具/美术工具合集` 打开主工具窗口。

> 包名在 `package.json` 中为 `com.yuxuhai.arttools`，你可以根据团队规范自行修改。

### 方式二：嵌入式（本地包）引用

如果你不想使用 Git URL，可以在目标项目中以“本地包”的形式引用：

1. 将当前仓库中的 `UnityArtTool/UnityArtTool` 整个文件夹复制或拷贝到目标项目的 `Packages` 目录下，例如：  
   - `YourUnityProject/Packages/com.yuxuhai.arttools`
2. 确认该目录下包含 `package.json`、`ArtTools.Editor.asmdef` 以及 `Managers/`、`ToolItems/` 等子目录。
3. Unity 会自动识别为一个本地包，并在 Package Manager 中显示出来。
4. 在菜单栏通过 `工具/美术工具合集` 打开主工具窗口。

### 方式三：传统拷贝到 Assets（不通过包管理器）

1. 将本工具所在的 `UnityArtTool/UnityArtTool` 目录拷贝到项目的 `Assets` 下的任意 Editor 目录中（确保脚本位于 Editor 代码环境中编译）。
2. 打开 Unity 后，等待脚本编译完成。
3. 在菜单栏中通过 `工具/美术工具合集` 打开主编辑器窗口。

> 菜单路径来源于 `TestToolsEditorWindow` 中的  
> `[MenuItem("工具/美术工具合集", false, 2)]`

## 快速上手（3 分钟完成基础配置）

### 第一步：创建配置文件

1. 在 Project 窗口空白处右键：  
   `Create -> ArtTools -> Test Tool Data`
2. 为配置文件起一个易懂的名字，例如 `ArtTools_Default`
3. 建议放在团队统一约定的目录，如：`Assets/ArtTools/Configs`

### 第二步：打开工具窗口并选择配置

1. 在菜单栏点击：`工具 / 美术工具合集`
2. 在窗口顶部的 **配置文件** 下拉框中，选择刚刚创建的 `Test Tool Data` 资产

### 第三步：进入编辑模式

1. 查看窗口底部，点击 `编辑模式` 按钮，进入配置状态
2. 左侧区域可以：
   - 新建标签页（输入名称后点击 `+`）
   - 通过勾选“重命名模式”来修改 Tab 名字
   - 删除不需要的标签页
   - 拖拽改变标签页顺序

### 第四步：添加工具项

1. 选中一个标签页
2. 在右侧 `添加新工具项` 区域：
   - 选择工具类型（例如 `Scene`、`OpenWeb`、`Tool` 等）
   - 填写对应配置（如下所示）
   - 点击右侧 `+` 按钮，将该工具项添加到当前标签页

常见配置示例：

- 打开场景：  
  - 工具类型：`Scene (OpenSceneAsset)`  
  - 选择对应场景资产，按钮名称会自动填为场景名  

- 打开网页：  
  - 工具类型：`OpenWeb`  
  - 填写按钮名称（例如：项目 Wiki）  
  - 填写 URL（例如：`https://confluence.xxx.com/...`）  

- 执行菜单命令：  
  - 工具类型：`Tool`  
  - 在下拉列表里选择 Unity 菜单（如 `Window/General/Hierarchy` 或项目自定义菜单）  

### 第五步：保存并使用

1. 当配置发生改变时，底部“保存配置”按钮会以 `保存配置 *` 的形式高亮提示
2. 点击 `保存配置` 将当前配置写回 `Test Tool Data` 资产
3. 关闭“编辑模式”，开始像普通工具栏一样使用所有按钮

## 界面结构说明

工具窗口整体分为四个区域：

1. **顶部区域**
   - 配置文件选择下拉框
   - 快速搜索折叠栏：输入关键词，选择搜索引擎，一键打开浏览器

2. **左侧标签页列表**
   - 以列表形式展示所有标签页
   - 编辑模式下可：
     - 新建标签页
     - 重命名标签页
     - 删除标签页
     - 拖拽调整标签顺序

3. **右侧工具项区域**
   - 上方：`添加新工具项` 面板（仅在编辑模式下显示）
   - 下方：当前标签页下的所有工具项列表  
   - 在编辑模式中，可拖拽改变工具项的排列顺序

4. **底部操作栏**
   - `联机启动` 按钮：使用配置中的“联机启动场景”一键启动并自动 Play
   - 联机场景、日志数量、截图路径等全局设置（编辑模式下可见）
   - `定位配置文件`：在 Project 窗口中选中当前使用的配置资产
   - `编辑模式`：切换编辑 / 使用两种状态
   - `保存配置`：保存当前所有修改

## 内置工具项类型详细说明

### ToolAsset（执行 Unity 菜单命令）

- 作用：把任意 Unity 菜单（包括你们项目自定义的 Editor 菜单）做成一键按钮。
- 典型用法：
  - 一键打开某个工具窗口
  - 一键执行打包 / 构建 / 烘焙等菜单

### FindObjectAsset（定位 Project 资源）

- 作用：在 Project 窗口中定位到指定资源（贴图、预制体、材质、脚本等）。
- 典型用法：
  - 快速跳转到“常用资源目录”或关键资源文件

### FindGameObjectAsset（查找场景中物体）

- 作用：在当前场景中查找指定名称的 GameObject，可限定所属场景。
- 典型用法：
  - 快速定位关键 UI、角色、光源、摄像机等物体

### TextAsset（文本说明）

- 作用：显示一段自定义文字，适合作为小标题、分组说明或操作提示。
- 典型用法：
  - 在工具列表中插入“操作说明”、“注意事项”等提示文字

### OpenPathAsset（打开系统文件夹）

- 作用：在系统文件管理器中打开一个目录。
- 典型用法：
  - 打开截图输出目录、打包输出目录、日志目录等

### OpenSceneAsset（打开场景）

- 作用：一键切换到指定场景，切换前会提示是否保存当前场景修改。
- 典型用法：
  - 快速切换到主城 / 战斗 / 登录 / Demo 等常用场景

### OpenWebAsset（打开网页）

- 作用：在系统默认浏览器或 Unity 内置浏览器中打开网页。
- 典型用法：
  - 打开项目 Wiki、Bug 系统、配置后台、监控平台等

### SeparatorAsset（分隔符）

- 作用：为空白间距或带标题的分隔线，用于视觉分组，提高可读性。

## 常见问题（FAQ）

**Q：在 Package Manager 里添加 Git URL 报 “No package manifest was found”？**  
A：请确认使用的地址为：  
`https://github.com/yuxuhai/UnityArtTool.git`  
并确保网络正常、GitHub 可访问；仓库中已包含 `package.json`（仓库已经配置好，一般只需保持网络正常即可）。

**Q：安装后找不到菜单“工具 / 美术工具合集”？**  
A：请确认：  
- 项目脚本已编译完成（等待 Unity 右下角进度条结束）  
- 工具脚本位于 Editor 代码环境（通过 UPM 安装会自动正确配置）  
- 菜单路径为：`工具 / 美术工具合集`

**Q：点击某个按钮没有反应怎么办？**  
A：可能的原因：
- 对应的菜单路径、场景、URL 已被修改或删除  
- 资源引用丢失（如场景被移动并重新命名）  
建议在编辑模式下打开对应工具项，检查配置是否正确，必要时重新指定资源或路径。

**Q：团队协作时配置会不会丢？**  
A：所有配置都保存在 `Test Tool Data` 这个 ScriptableObject 资产里，建议：
- 将配置资产和脚本一起纳入版本管理（Git / SVN）  
- 在团队内约定统一的保存位置（例如 `Assets/ArtTools/Configs`）
